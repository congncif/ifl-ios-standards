# Process docs index

Use these process docs to decide how work is planned, documented, approved, written, and reviewed.

- `docs-organization.md` — where project documents live and how they are named/lifecycled.
- `long-document-writing.md` — chunk-safe writing for generated plans, specs, reports, and handoffs.
- `requirement-intake.md` — requirement summary, open-question classification, and Stage 1 gate behavior.
- `approval-modes.md` — co-working vs auto approval gates and reviewer verdicts.
- `full-auto-operating-model.md` — eligibility, authority, independent gates, recovery/resume, frozen
  candidate identity, finding disposition, and engineering-completion boundary.
- `lean-verification.md` — plan-scale execution, code-only testing, semantic commits, and one final AI consistency review.
