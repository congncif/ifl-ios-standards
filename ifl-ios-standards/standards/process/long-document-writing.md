# Process — Long document writing

Use for long plans, standards chapters, ADRs, migration guides, or reports.

## Writing pattern

1. Choose the durable location from `process/docs-organization.md`.
2. Create a short outline when the document has more than five major sections.
3. Write one coherent major section at a time.
4. Keep terminology and decisions consistent with the approved plan and linked source documents.
5. Avoid copying source dumps, conversation history, receipts, hashes, or status machinery.
6. Write the conclusion/status after the content is complete.

Split by purpose before splitting by size. Requirements, plan, final AI review, and optional final
report are separate documents; agent assignments and intermediate findings are not documents by
default.

When multiple agents contribute, assign disjoint sections or serialize one shared document through an
integration owner. After all planned documents are complete, include them in the plan's single final
AI consistency review. Do not run a review or verification cycle after each section.
